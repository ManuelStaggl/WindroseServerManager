## Spacing Scale

Deklarierte Werte (Vielfache von 4) — konsistent mit dem bestehenden App-Layout:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-Abstände innerhalb von Feature-Tiles, PasswordBox-Innen-Abstände |
| sm | 8px | Abstände zwischen Buttons, Spacing in StackPanels (Wizard-Footer, Banner-Aktionen) |
| md | 16px | Card-Padding, Abstand zwischen Wizard-Schritten, Feature-Grid-Spacing |
| lg | 24px | Dialog-Padding (AboutDialog verwendet 24px — Wizard übernimmt), Seitenabstand |
| xl | 32px | Wizard-Fenster-Außenabstand wenn nötig |
| 2xl | 48px | nicht verwendet in Phase 9 |
| 3xl | 64px | nicht verwendet in Phase 9 |

> Quelle: ConfirmDialog.axaml (Padding 20px), AboutDialog.axaml (Padding 24px), DashboardView.axaml (Margin-Pattern), GlobalStyles.axaml (card Padding 16px).

Ausnahmen:
- Wizard-Stepper-Indikatoren: 12px Kreis-Durchmesser, 8px Gap zwischen Kreis und Label
- Feature-Tile-Icons: 32px Größe (größer als Standard-Icon 16px, da Feature-Showcase)
- Retrofit-Banner: 12px vertikales Padding (zwischen md und sm — Banner ist kompakter als eine Card)

## Komponenten-Inventar (Phase 9)

Alle neuen Komponenten folgen dem Pattern existierender Dialoge.

### 1. InstallWizardWindow (neues FluentWindow / Window)

- **Basis:** `Window` (wie ConfirmDialog, AboutDialog) mit `Background="{DynamicResource BrandNavySurfaceBrush}"`
- **Größe:** Width=560, SizeToContent=Height, CanResize=False
- **Struktur:** Outer Border (CornerRadius=8, Padding=24, BorderBrush=BrandNavyBorderBrush, BorderThickness=1) enthält:
  1. Stepper-Header (horizontal, 3 Schritte)
  2. Step-Content-Zone (wechselt je Schritt)
  3. Footer (Cancel links, Weiter/Bestätigen rechts)

**Stepper-Header:**
- 3 Kreise (Diameter 12px) verbunden durch 1px Linie in BrandNavyBorderBrush
- Aktiver Kreis: BrandAmberBrush-Hintergrund, weiße Schrift
- Abgeschlossener Kreis: BrandSuccessBrush-Hintergrund (Checkmark-Glyph)
- Inaktiver Kreis: BrandNavySurfaceAltBrush, BrandTextMutedBrush-Schrift
- Step-Label: Classes="caption" unter jedem Kreis

**Schritt 1 — Installationspfad:**
- Übernimmt das bestehende Pfad-Input-Pattern aus InstallationView (TextBox + Browse-Button)
- Kein Fortschritts-Indikator in diesem Schritt

**Schritt 2 — WindrosePlus (Kern-Schritt):**
- `WindrosePlusOptInControl` UserControl (shared, auch in Retrofit verwendet)
- Enthält: Feature-Grid + Disclaimer + Secrets-Form
- Opt-in CheckBox ist standardmäßig angehakt (WIZARD-02)

**Schritt 3 — Bestätigung:**
- Zusammenfassung: Pfad, WindrosePlus-Status (Ja/Nein), Port
- Einzige Aktion: "Installieren" (primary) — startet SteamCMD + WindrosePlus-Install
- Fortschritts-Anzeige: IndeterminateProgressBar (wie InstallationView Zeile 30) + aktueller Phase-Text

### 2. WindrosePlusOptInControl (neues UserControl)

Wird sowohl im Wizard-Schritt 2 als auch im Retrofit-Dialog verwendet.

**Feature-Grid:**
- 2-Spalten-Grid mit 3 Zeilen = 6 Tiles (Kick/Ban — Players — Broadcast — Events — Sea-Chart — INI-Editor)
- Jede Tile: Border mit CornerRadius=8, Background=BrandNavySurfaceAltBrush, Padding=12, Spacing=6 (Icon + Label)
- Icon: 32px PathIcon oder TextBlock mit Emoji-Symbol in BrandAmberBrush
- Label: Classes="body" (14px Regular)

**Disclaimer:**
- Classes="caption" + BrandTextMutedBrush
- Text: "WindrosePlus ist ein separates MIT-Mod. Es wird in die Server-Dateien installiert und kann jederzeit in den Einstellungen deaktiviert werden."

**GitHub-Link:**
- Classes="subtle" Button: "WindrosePlus von HumanGenome · MIT · Mehr erfahren ↗"
- Tag = "https://github.com/HumanGenome/WindrosePlus", Click = OpenUrl

**Opt-in CheckBox:**
- Standard: IsChecked=True
- Label: "WindrosePlus installieren" (Classes="body")
- Steuert die Sichtbarkeit der Secrets-Form (IsVisible="{Binding IsOptingIn}")

**Secrets-Form (nur sichtbar wenn opt-in aktiv):**
- RCON-Passwort: PasswordBox (masked) mit Eye-Toggle + Copy-Button + Regenerate-Button, inline in Grid
- Dashboard-Port: TextBlock (read-only, Classes="body") + Caption "Wird automatisch gewählt (Port 18080–18099)"
- Steam-ID: TextBox mit Watermark "SteamID64 oder steamcommunity.com/profiles/..."
  - Inline-Validierung on Blur: rote Border + Caption "Ungültiges Format — bitte numerische URL oder SteamID64 angeben"
  - Gültig: grüne Border (500ms Transition, dann zurück zu Normal)

### 3. Retrofit-Banner (in DashboardView)

- **Position:** Oberhalb aller bestehenden Cards, unterhalb des page-title-Blocks
- **Sichtbarkeit:** `IsVisible="{Binding RetrofitBannerVisible}"` (true wenn OptInState=NeverAsked)
- **Kein Close-X** — explizite Entscheidung erforderlich (RETRO-03)

**Layout:**
```
Border (card-Klasse + BorderBrush=BrandWarningBrush + BorderThickness=1 + Margin=0,0,0,16)
  Grid (ColumnDefinitions="*,Auto")
    [Col 0] StackPanel (Spacing=6)
      TextBlock (section-header) "WindrosePlus verfügbar"
      TextBlock (body muted) "Schalte Spieler-Management, Events, Seekarte und Konfigurations-Editor frei."
    [Col 1] StackPanel (Orientation=Horizontal, Spacing=8, VerticalAlignment=Center)
      Button (primary) "Jetzt installieren"  → öffnet RetrofitDialog
      Button (subtle)  "Nicht jetzt"         → setzt OptInState=OptedOut, Banner verschwindet
```

### 4. RetrofitDialog (neues FluentWindow / Window)

- **Basis:** Window, Width=540, SizeToContent=Height, WindowStartupLocation=CenterOwner
- **Background + Border:** wie ConfirmDialog (BrandNavySurfaceBrush, BrandNavyBorderBrush)
- **Inhalt:**
  1. `TextBlock Classes="section-header"` — "WindrosePlus installieren"
  2. `WindrosePlusOptInControl` (shared UserControl, opt-in CheckBox ebenfalls default-on)
  3. Footer-Buttons: "Abbrechen" (subtle, links) — "Installieren" (primary, rechts)

## Interaction Contract

### Wizard-Stepper-Navigation
- "Weiter" validiert den aktuellen Schritt vor dem Wechsel
- Schritt 1 → 2: Nur wenn Pfad nicht leer (Basis-Check; tiefe Validierung läuft im Hintergrund)
- Schritt 2 → 3: Wenn opt-in aktiv: Steam-ID muss format-valide sein. RCON-Passwort ist bereits vorab generiert.
- "Zurück"-Button (subtle) ist ab Schritt 2 sichtbar
- Während Install läuft (Schritt 3): alle Buttons disabled außer "Abbrechen" (wenn cancellable)

### RCON-Passwort-Feld
- Startet masked (PasswordChar="•")
- Eye-Toggle-Button (Icon: Auge, 32px quadratisch, subtle) rechts im PasswordBox-Container
- Copy-Button (Icon: Clipboard, 32px quadratisch, subtle) rechts neben Eye-Toggle
- Regenerate-Button (Icon: Pfeil-Kreis, 32px quadratisch, subtle) ganz rechts
- Alle drei Buttons in einer horizontalen Gruppe, Spacing=4px

### Steam-ID-Validierung
- Trigger: on Blur (Fokus verlassen)
- Regex mit `TimeSpan.FromSeconds(5)` Timeout (Coding-Standard)
- Valid: `^(765611\d{11}|https?://steamcommunity\.com/profiles/765611\d{11}/?)$`
- Bei Ungültig: rote Border (BrandErrorBrush, 2px) + Caption in BrandErrorBrush darunter
- Bei Gültig: normale Border (BrandNavyBorderBrush)

### Retrofit-Banner Lifecycle
- Erscheint: wenn `RetrofitBannerVisible == true` (OptInState=NeverAsked)
- "Jetzt installieren" → öffnet RetrofitDialog (modal, Owner=MainWindow)
  - Bei Confirm: Install läuft, auf Success → Banner verschwindet, Toast "WindrosePlus installiert"
  - Bei Cancel: Banner bleibt (State=NeverAsked)
- "Nicht jetzt" → OptInState=OptedOut, Banner verschwindet permanent für diesen Server

### Progress-States (Wizard Schritt 3 + Retrofit-Dialog während Install)
- IndeterminateProgressBar (4px Höhe, BrandAmberBrush-Farbe via SystemAccentColor)
- TextBlock (caption) zeigt aktuellen `InstallProgress`-Stage-Text
- Alle primären Buttons disabled während Install
- Bei Fehler: Toast (BrandErrorBrush) + Fehlermeldung inline unter der ProgressBar

### Feature-Tile Hover
- Border-Farbe wechselt zu BrandAmberBrush, 150ms Transition
- Kein Click-Handler — rein informativ

### Animationen
- Wizard-Schritt-Wechsel: Opacity 0→1, 200ms, CubicEase — wie BasePage-Pattern
- Banner erscheint: kein Fade (da beim Navigieren zur Seite schon sichtbar)
- Retrofit-Dialog öffnet: WindowStartupLocation=CenterOwner, kein eigener Fade nötig (Avalonia Window-Standard)
